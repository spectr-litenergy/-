

# НутриЛог — дневник питания с подсчётом КБЖУ

**НутриЛог** — это простое и удобное веб-приложение для ведения персонального дневника питания. Оно помогает отслеживать потребление калорий, белков, жиров и углеводов, вести собственную базу продуктов и анализировать рацион за любой день.

## Возможности

- **База продуктов** — добавляйте, редактируйте и удаляйте продукты с указанием КБЖУ на 100 грамм.
- **Три приёма пищи** — завтрак, обед и ужин с возможностью добавлять продукты и указывать вес порции.
- **Автоматический пересчёт** — КБЖУ автоматически пересчитываются пропорционально весу порции.
- **Статистика за день** — суммарные калории, белки, жиры и углеводы с прогресс-баром относительно дневной цели (2000 ккал).
- **История питания** — просматривайте записи за любой день с помощью удобной навигации.
- **Автосохранение** — все данные сохраняются в localStorage браузера и не теряются после перезагрузки.
- **Адаптивный дизайн** — приложение одинаково хорошо выглядит на компьютере, планшете и смартфоне.
- **Интуитивный интерфейс** — все операции выполняются в 2-3 клика через модальные окна.

## Использование

### База продуктов
- **Добавить продукт**: нажмите кнопку «Добавить продукт», заполните поля (название, калории, белки, жиры, углеводы) и нажмите «OK».
- **Редактировать**: нажмите на иконку карандаша рядом с продуктом, измените данные и сохраните.
- **Удалить**: нажмите на иконку корзины и подтвердите удаление.

### Дневник питания
- **Добавить в завтрак/обед/ужин**: нажмите кнопку «Добавить в завтрак» под соответствующим блоком, выберите продукт и укажите вес порции.
- **Быстрое добавление**: в таблице продуктов нажмите на иконку ложки, выберите приём пищи и вес.
- **Удалить запись**: нажмите на иконку корзины рядом с записью в блоке приёма пищи.

### Навигация по дням
- Используйте кнопки «<» и «>» в верхней панели для переключения между днями.
- Все изменения автоматически сохраняются за соответствующий день.

### Статистика
- В правой панели отображаются итоговые значения за день и прогресс-бар относительно цели 2000 ккал.

## Тестирование

Для проверки работоспособности приложения можно использовать следующие тестовые сценарии:

- Добавление нового продукта с валидными данными.
- Редактирование существующего продукта.
- Удаление продукта с подтверждением и отменой.
- Добавление продукта в завтрак, обед, ужин с разным весом.
- Удаление записи из приёма пищи.
- Проверка суммирования КБЖУ за день.
- Переключение между днями с сохранёнными данными.
- Обновление страницы и проверка сохранения данных.
